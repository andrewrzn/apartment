# UI Guidelines

Единые правила визуального стиля для всех текущих и будущих тренажеров.

## 1) Цвета и фон

- Базовый фон приложения: мягкий, светлый (`#f1f7f2` или близкий).
- Акцентные цвета:
  - синий — основной прогресс/нейтральные действия,
  - зеленый — прогресс по уровню/позитивные маркеры,
  - красный — предупреждения/важные CTA, когда уместно.
- Избегать слишком насыщенных кислотных оттенков.

## 2) Типографика

- Заголовок страницы (`h1`): крупный, контрастный, центрированный на стартовой.
- Подзаголовки секций: `st.subheader` + короткая подпись.
- Основной текст: читаемый, контрастный, без бледно-серого.
- Не использовать `Ур.1/2/3`; только `Уровень 1/2/3`.

## 3) Карточки и отступы

- Карточки направлений: одинаковая высота и структура.
- Кнопка карточки: `Открыть направление`.
- Основные интервалы:
  - между секциями: 12–16 px,
  - внутри карточек: 10–14 px.
- После блоков статистики использовать визуальный разделитель (`---`), если дальше идет другой тип контента.

## 4) Кнопки и состояния

- На стартовой у карточек допустимы разные цвета кнопок, но единый текст.
- Кнопки действий внутри задач:
  - `Проверить ответ` (primary),
  - `Показать ответ` (secondary).
- При неверном ответе не показывать решение автоматически.

## 5) Статистика и прогресс

- Всегда разделять:
  - статистику за сеанс (`attempts/success/%`),
  - прогресс по задачам уровня (`solved/total`).
- Эти два прогресса должны быть визуально различимы (цвет/подпись).

## 6) Чертежи

- Все чертежи внутри одного модуля должны иметь одинаковую визуальную рамку:
  - единый масштаб осей,
  - единый размер вывода (например, `width=520`),
  - одинаковые толщины линий и контраст подписей.

## 7) Библиотека материалов

- Название раздела: `Библиотека`.
- Структура файлов: `archive_pdfs/<тема>/<подтема>/<file>.pdf`.
- В UI: выбор темы и подтемы + кнопки скачивания.
