# AGENTS.md

Правила для ассистентов/агентов, которые вносят изменения в проект.

## Главные принципы

- Сохранять единый UX: приложение должно выглядеть как один продукт.
- Не ломать существующие разделы при добавлении новых тем.
- Для новых тем использовать модульный подход: отдельный `*_module.py` + подключение в `app.py`.
- Перед завершением изменений проверять:
  - `python -m py_compile ...`
  - базовую визуальную целостность экранов.

## Структура разделов

- `app.py` — единая точка входа и стартовая навигация.
- `problems_catalog.py` — каталог задач «Геометрия квартиры».
- `proporsii_trenajer_module.py` — модуль тренажера пропорций.
- `archive_pdfs/` — библиотека PDF по схеме `тема/подтема/файлы`.

## UI-правила (кратко)

- Единый мягкий фон приложения (`.stApp`) без резких контрастов.
- Заголовки и подписи — контрастные, не бледно-серые.
- Кнопки действий — понятные и единообразные по формулировкам.
- Прогрессы должны быть семантически различимы:
  - статистика за сеанс (синий),
  - прогресс уровня (зеленый).
- Чертежи должны иметь унифицированный размер/масштаб в пределах одного модуля.

## Тексты и термины

- Использовать: `Уровень 1`, `Уровень 2`, `Уровень 3` (не `Ур.1`).
- Использовать термин `Библиотека` (не `Архив` в UI).
- Стартовая карточка: кнопка `Открыть направление`.

## Добавление новой темы (например, скорость)

1. Создать отдельный модуль темы, например `skorost_trenajer_module.py`.
2. Описать задачи в структуре данных (условие, ответ, объяснение, при необходимости чертеж).
3. Подключить модуль в `app.py` как новое направление на стартовой странице.
4. При необходимости добавить экспортный скрипт по аналогии с `export_workbook.py`.
5. Убедиться, что стили соответствуют `UI_GUIDELINES.md`.
